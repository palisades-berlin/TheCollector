# screen-collector — Claude Context

## What This Is
Chrome Extension (Manifest V3) that captures full-page screenshots in one click. No build step — plain ES modules loaded directly by Chrome. All storage is local (IndexedDB), no network calls.

## Project State
MVP + History page complete. Missing features planned for future passes:
- PDF export (offscreen.js already structured for it)
- Options page (auto-download toggle, PDF page size)
- Multi-image fallback for pages > 16 000 px tall
- Premium stubs (crop, annotations, URL/timestamp overlay)

## Architecture

### Contexts & Their Roles

| Context | File | What it does |
|---|---|---|
| Service Worker | `src/background/service-worker.js` | Orchestrates the entire capture: injects content script, runs the scroll/capture loop, calls `captureVisibleTab`, sends tiles to offscreen, opens preview tab |
| Content Script | `src/content/capture-agent.js` | Injected on demand. Measures page, suppresses fixed elements, scrolls on command, signals when settled |
| Offscreen Document | `src/offscreen/offscreen.js` | Has DOM/Canvas access. Stitches tiles into one image, saves to IndexedDB |
| Popup | `src/popup/popup.js` | Triggers capture, shows progress ring, listens for SW broadcasts |
| Preview Tab | `src/preview/preview.js` | Loads screenshot from IDB, zoom toggle, PNG/JPG download |

### Message Flow
```
Popup ──CAPTURE_START──► Service Worker
                              │
                    CS_GET_METRICS ──► Content Script ──► {scrollW,scrollH,viewW,viewH,dpr}
                    CS_PREPARE    ──► Content Script  (hide fixed/sticky elements)
                    [loop]
                      CS_SCROLL_TO ──► Content Script ──► {done:true}
                      captureVisibleTab() → dataUrl
                    [end loop]
                    CS_RESTORE    ──► Content Script  (restore + scroll back)
                              │
                    OS_STITCH ──► Offscreen Document (stitch canvas → save IDB)
                              │
                    chrome.tabs.create(preview.html?id=...)
                              │
                    SW_PROGRESS / SW_DONE / SW_ERROR ──► Popup (broadcast)
```

### Message Types
All defined in `src/shared/messages.js` as `MSG.*`.
The content script is a self-contained plain JS IIFE — it uses string literals for message types (not imports). Keep them in sync manually when changing `messages.js`.

## Key Files

- `src/shared/messages.js` — single source of truth for all message type strings
- `src/shared/constants.js` — `MAX_CANVAS_SIDE` (16 000px), `SCROLL_SETTLE_MS` (200ms)
- `src/shared/db.js` — IndexedDB helpers: `saveScreenshot`, `getScreenshot`, `listScreenshots`, `deleteScreenshot`, `saveTile`, `getTiles`, `deleteTiles`. DB name: `screen-collector`, store: `screenshots`
- `manifest.json` — MV3, no build output needed, service worker at `src/background/service-worker.js`
- `create_icons.py` — pure-Python PNG generator for `assets/icons/`. Run after any icon/palette change.

## Conventions

- **No bundler.** Files are ES modules loaded by Chrome directly. Use `type="module"` in HTML script tags and `"type": "module"` in the manifest background config.
- **Content script is a plain IIFE.** Cannot use ES module imports. All logic must be self-contained in `capture-agent.js`. Guard double-injection with `if (window.__screenCollectorAgent) return;`.
- **Offscreen document** is created lazily via `ensureOffscreen()` before the stitch step. Chrome only allows one offscreen document per extension at a time.
- **No framework.** Vanilla JS only — no React, Vue, etc.
- **No CDN dependencies.** Any third-party libs must be vendored in `vendor/`.
- **`captureVisibleTab` only works from the service worker**, not from content scripts. Always call it from `service-worker.js` passing `tab.windowId`.
- **Broadcast vs. response:** SW broadcasts progress via `chrome.runtime.sendMessage` (fire-and-forget, `.catch(() => {})`). The offscreen stitch returns a proper `sendResponse`. Popup listeners never call `sendResponse`.

## IndexedDB Schema
```
DB: 'screen-collector'  (v2)

Store: 'screenshots'  keyPath: 'id'
  id:        string  (UUID)
  url:       string  (source page URL)
  title:     string  (source page title)
  timestamp: number  (Date.now())
  blob:      Blob    (PNG image data)
  width:     number  (px)
  height:    number  (px)

Store: 'pending_tiles'  keyPath: 'id'  (temporary capture buffer)
  id:     string  ("${jobId}:${index}")
  jobId:  string  index — same UUID as the screenshot id
  index:  number  (tile sequence: row*cols + col)
  x:      number  (pixel x position in final image)
  y:      number  (pixel y position in final image)
  blob:   Blob    (PNG tile data)
```

**Why pending_tiles exists:** `chrome.runtime.sendMessage` has a hard 64 MiB limit. Sending all tile dataURLs at once in a single message fails on long/high-DPI pages. Instead, the SW converts each `captureVisibleTab` dataURL to a Blob via `fetch(dataUrl).then(r => r.blob())` and writes it to `pending_tiles`. The OS_STITCH message then only carries tiny metadata (`id`, `totalW`, `totalH`, `sourceUrl`, `title`). The offscreen document reads the tiles back from IDB using `getTiles(jobId)`, stitches them, and calls `deleteTiles(jobId)` to clean up.

## Verification (No Dev Server)
The extension must be loaded in Chrome, not a dev server.

**Load:** `chrome://extensions` → Developer mode → Load unpacked → `screen-collector/`

**Static UI check** (for CSS/layout changes only):
- Start `Asset Preview Server` (Python http.server on port 8765 from `ClaudeCode/`)
- Visit `http://localhost:8765/screen-collector/src/popup/popup.html`
- Visit `http://localhost:8765/screen-collector/src/history/history.html`
- Visit `http://localhost:8765/screen-collector/src/preview/preview.html?id=test`
- Chrome APIs will fail (expected) — verify layout/styles only

**Full functional test:**
1. Load unpacked in Chrome
2. Navigate to a long page (e.g. `en.wikipedia.org/wiki/JavaScript`)
3. Click icon → verify progress bar animates
4. Preview tab opens → scroll to verify full page captured
5. Click image → verify zoom toggle
6. Download PNG → verify file opens at full resolution
7. Test `Alt+Shift+P` shortcut on another page
8. Click History in popup → verify grid of past captures, open/delete work

## Common Gotchas
- After editing any file, manually click the refresh icon on `chrome://extensions` for the service worker to pick up changes
- The offscreen document caches `offscreenReady = true` in the SW — if the SW is killed and revived, `getContexts()` re-checks before creating
- `chrome.tabs.captureVisibleTab` requires the tab to be the active tab in its window; keyboard shortcut captures whichever tab is currently active
- Canvas > 16 000px on any side throws an error (intentional, multi-image fallback is a TODO)
